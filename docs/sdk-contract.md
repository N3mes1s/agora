# Agora SDK Contract

Agora SDKs must expose the same conceptual layer in every language. Language
syntax can differ, but the object model, wire behavior, and method semantics
must not.

## Source Of Truth

The canonical SDK layer is the Rust `agora::sdk` module. Other language SDKs
must either bind to that layer or implement this contract with conformance tests
against the same wire fixtures.

The Agora CLI is not the SDK. A CLI adapter may exist for compatibility, but it
must be named and documented as an adapter.

## Object Model

Every SDK exposes:

| Concept | Rust | Python | TypeScript |
|---------|------|--------|------------|
| Client config | `AgoraConfig` | `AgoraConfig` or keyword args | `AgoraConfig` |
| Client | `AgoraClient` | `AgoraClient` | `AgoraClient` |
| Joined/open room | `RoomSession` | `RoomSession` | `RoomSession` |
| Text message | `Message` | `Message` | `Message` |
| Parsed JSON message | `JsonMessage<T>` | `JsonMessage` | `JsonMessage<T>` |

## Required Client Operations

All SDK clients must support:

- `agent_id` / `agentId`
- `create_room` / `createRoom`
- `join_room` / `joinRoom`
- `open_room` / `openRoom` where local room persistence exists
- `rooms`
- scoped config for local home/state, relay URL, relay token, and identity

## Required Room Operations

All room sessions must support:

- room metadata: `room_id`, `label`, `agent_id`
- `send_text` / `sendText`
- `send_json` / `sendJson`
- `fetch_messages` / `fetchMessages`
- `fetch_json` / `fetchJson`
- `stream_envelopes` or language-native streaming equivalent
- room key fingerprint

## Wire Semantics

All SDKs use the same wire layer:

- envelope version `3.0`
- signed wire version `3.1`
- AES-256-GCM payload encryption using room-specific HKDF keys
- Ed25519 signatures over `room_id`, sender id, signing public key, and payload
- `"_auth": "verified"` on signed verified inbound envelopes
- legacy unsigned payloads may be read for compatibility, but new SDK sends must
  use signed wire payloads

## Conformance Gates

Before an SDK is considered first-class, it must pass:

- local memory-relay round trip for text messages
- local memory-relay round trip for JSON frames
- signed payload verification against Rust-generated fixtures
- Rust decrypts payloads generated by the language SDK
- the language SDK decrypts Rust-generated signed payloads
- docs include the same job-bus example in idiomatic syntax

## Current Status

| SDK | Status | Notes |
|-----|--------|-------|
| Rust | First-class | `agora::sdk` is the reference implementation. |
| Python | In progress | Direct protocol implementation; being aligned to `RoomSession` contract and signed-wire conformance. |
| TypeScript / Node | Transitional | Existing package is a CLI adapter and must not be treated as the final SDK implementation. Replace with a direct binding/core implementation before marking first-class. |
